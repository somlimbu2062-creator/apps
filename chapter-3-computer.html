Chapter 3: Computer Software & Operating System

A guide to the programs that make computers work.

Introduction to Software

A **computer software** is a collection of related programs and documents that instruct the computer hardware on what to do. Hardware alone is just a set of physical components; software is what brings it to life and makes it useful. Without software, hardware cannot function, and without hardware, software cannot be utilized. There are two main types of software: System Software and Application Software.

Types of Software

1. System Software

System software is designed to control and manage the overall operations of a computer system. It acts as an intermediary between the user and the computer hardware, supporting the running of other software and managing system resources. It is categorized into two main types:

  • System Management: These programs are responsible for the proper functioning of the computer. Examples include **Operating Systems**, **Utility Software** (e.g., antivirus, disk cleanup), and **Device Drivers**.
  • System Development: This software is used by programmers to develop new programs. Examples include **Programming Languages** (e.g., Python, C++) and **Language Translators** (e.g., compilers, interpreters).

2. Application Software

Application software is designed to perform specific tasks for the user. These programs fulfill user-oriented problems, such as preparing bills, creating documents, or editing photos. They are often developed using high-level programming languages.

  • Packaged Software: These are generalized, off-the-shelf programs designed for a broad audience. They are often large, advanced, and developed by reputable companies (e.g., Microsoft Office, Adobe Photoshop).
  • Customized (Tailored) Software: This software is developed specifically for a particular user or organization to perfectly match their unique requirements (e.g., payroll systems, school management software).

Operating System (OS)

An **Operating System (OS)** is a core set of programs that manages all the hardware and software resources of a computer. It provides a platform for other applications to run and acts as an essential interface between the user and the computer hardware. The primary goal of an OS is to ensure the computer system operates efficiently and smoothly.

Key Functions of an OS

  • Input-Output Management: Manages communication between the computer and peripheral devices.
  • User Interface (UI): Provides a way for users to interact with the computer. This can be a **Character User Interface (CUI)** like MS-DOS or a **Graphical User Interface (GUI)** like Windows.
  • Security: Protects the system from unauthorized access and threats through features like user accounts and passwords.
  • Process Management: Manages the execution of different programs and tasks.
  • Memory Management: Allocates and deallocates memory to programs as needed.
  • Data Management: Organizes data into logical groupings called files and directories.
  • Deadlock Prevention: Ensures that two processes don't get stuck waiting for each other to release resources.

Types of Operating Systems

A) Based on Processing

  • Multi-programming OS: Allows multiple programs to be in the computer's memory at the same time, with a single CPU executing them concurrently.
  • Multi-tasking OS: Allows a single user to run more than one program at the same time (e.g., listening to music while typing a document).
  • Multi-processing OS: Uses more than one CPU to execute multiple programs simultaneously, dramatically increasing processing speed.
  • Time-sharing OS: Allows multiple users on different terminals to use the same computer system at the same time by sharing the CPU's processing time.
  • Batch Processing OS: Groups similar jobs together in a "batch" and processes them without user interaction.
  • Real-time OS: Designed for systems where tasks must be completed within a strict time frame (e.g., rocket launching, airline reservations).

B) Based on User Interface

  • Character User Interface (CUI): Users interact with the computer by typing commands. It is faster but requires the user to remember commands (e.g., MS-DOS).
  • Graphical User Interface (GUI): Users interact with the computer using visual elements like icons, windows, and menus. It is very user-friendly but requires more memory (e.g., Windows, macOS).

Key OS Terminologies

  • Spooling: A technique to manage the difference in speed between the fast CPU and slow I/O devices, especially in printing.
  • Process: A program in execution. It is dynamic, unlike a static program file.
  • Thread: A smaller sub-process within a main process. Multi-threading allows parts of a single program to run concurrently.
  • Scheduling: The process by which the OS decides which program to run next, using algorithms like **Priority Scheduling** or **Time-sharing Scheduling**.
  • Buffering: Using a temporary storage area (buffer) to hold data during I/O operations, helping to synchronize data flow.
  • Virtual Memory: A technique that uses a portion of the hard disk to temporarily store data from RAM, allowing the computer to run large programs even with limited physical memory.
