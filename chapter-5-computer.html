Chapter 5: Programming Concepts and Logics

Introduction

As we know, every computerized device does not work by itself. They need to be given some sort of command that makes the computer work. These commands help to instruct the computer and program a specific action required by the user. Since the commands are given in the form of a computer language, we need programming languages to write the set of instructions which command the computer to perform certain actions, collectively called **programs**.

A program written in any programming language is not understood by the computer. Hence, we need some sort of language translator or processor, such as an **assembler**, **compiler**, and **interpreter**, which has the basic function of converting a program written in any type of language into **machine level language**.

The program written by the programmer is known as the **source program**. After converting it becomes an **object program**.

Qualities of a Good Program

  • It should be easily understood.
  • A program should be correct; it should be error-free.
  • It should be reliable.
  • It should have an easily understandable user interface.
  • It should be portable and flexible.

Programming Language

In order to make communication between a user and a computer, we need a computer language that helps a user to generate commands to perform as per the requirement. This language, with which we can give instructions to the computer, is known as a **programming language**.

The process of writing a programming language is known as **programming**, and the person who writes programs is a **programmer**.

Types of Programming Language

There are several types of programming languages which can be categorized as follows:

1. Low-Level Language

Low-level languages are machine-dependent languages, which means a program written for one type of system cannot be run on another. Programming in a low-level language is very difficult and time-consuming. Different types of low-level languages are:

a. Machine-Level Language (1GL)

This language consists of a sequence of 0's and 1's to generate instructions. It has a higher execution speed as it is directly understood by the processor.

Advantages and Disadvantages
  • **Advantages:** Directly understood by the processor, no translator needed.
  • **Disadvantages:** Difficult to develop and debug, machine-dependent.

b. Assembly Language (2GL)

In this language, instead of writing instructions in a series of 0's and 1's, we can use **mnemonics** (symbolic instructions) like `ADD`, `SUB`, etc. An **assembler** is needed to convert it into machine-level language.

Advantages and Disadvantages
  • **Advantages:** Easier to write and debug than machine language, faster execution than high-level languages.
  • **Disadvantages:** Machine-dependent, complex mnemonics, difficult and time-consuming.

2. High-Level Language (HLL)

This language is close to English and uses mathematical notations. It's a machine-independent language, making it easier to write and debug. A **compiler** or **interpreter** is needed to convert it into machine language. **FORTRAN** (Formula Translator) was the first high-level language.

Advantages and Disadvantages
  • **Advantages:** Easier to write, debug, and understand; machine-independent.
  • **Disadvantages:** Slower execution speed, requires a language translator.

a. Procedural-Oriented Language (3GL)

This language focuses on the procedure rather than the data. It follows a top-to-bottom approach and has less data security. Examples include C, FORTRAN, and QBasic.

Advantages and Disadvantages
  • **Advantages:** Easier to debug than low-level languages, machine-independent, general-purpose.
  • **Disadvantages:** Slower execution, less data security.

b. Problem/Object-Oriented Language (4GL)

This is an advanced form of HLL that focuses on data rather than procedures. It follows a bottom-up approach and has features like data encapsulation and inheritance, making data more secure. Examples include C#, C++, and Java.

Advantages and Disadvantages
  • **Advantages:** Can develop advanced, user-friendly software and web-based applications, is machine-independent.
  • **Disadvantages:** Slower execution, requires a language translator.

c. Natural Language (5GL)

This language uses simple statements from common communication languages. It is still in the developmental stage. An example is **PROLOG** (programming logic).

Advantages and Disadvantages
  • **Advantages:** Easier to develop and debug, machine-independent.
  • **Disadvantages:** Still in development, slower execution.

Language Translator/Processor

A language translator is a system software that converts a source program (assembly or high-level language) into a machine-understandable language (object program).

Assembler

Converts assembly-level language into machine language at once. It's relatively fast since assembly language is closer to machine code.

Compiler

Converts the entire program written in a high-level language into machine code at once. It finds errors after compiling the complete program, making debugging complex and time-consuming. Languages like C and C++ use compilers.

Interpreter

Translates a high-level language program one statement at a time. This makes debugging easier and less time-consuming, but the program execution is slower than with a compiler. Languages like BASIC, C#, and PHP use interpreters.

S.N. Compiler Interpreter
1 Translates the whole program at once. Translates one line at a time.
2 Finds errors after compiling the complete program. Finds errors after translating a single line.

4 Debugging is complex and difficult. Debugging is easier.

Errors (Bugs)

An error in a program is known as a **bug**. The process of removing errors is called **debugging**. There are three types of errors:

1. Syntax Error

An error that arises from not following the proper rules or format of the programming language. These are detected by a language processor. For example, in C, if a statement doesn't end with a semicolon, it's a syntax error.

2. Logical Error

An error due to poorly developed logic. This type of error is not detected by a language processor. For example, using the wrong formula to calculate simple interest would be a logical error.

3. Runtime Error

An error that occurs while the program is running. It is also not detected by a language translator. An example is insufficient memory space or a peripheral device not being turned on.

Common Terminology

  • **Data Types:** Specifies the type of data to be used, such as **Numeric** (whole numbers) and **Character** (alphanumeric values).
  • **Variable and Constant:** A **variable** can change its value during program execution, while a **constant** does not.
  • **Operands and Operators:** In `S = A + B`, `S`, `A`, and `B` are **operands**, and `+` and `=` are **operators**.
  • **Program Design Tools:** Includes **Algorithm**, **Flowchart**, and **Pseudocode**. An **algorithm** is a step-by-step procedure for solving a problem. A **flowchart** is a pictorial representation of these steps.

C Programming Language

A powerful high-level language developed by Dennis Ritchie at Bell Laboratories in 1970. It is often called a **Middle-Level Programming Language** because it supports both low-level and high-level programming.

Key Features of C Language

  • It supports structured programming.
  • It is portable.
  • It has powerful tools and keywords.
  • It is widely used for system programming.
Advantages and Disadvantages
  • **Advantages:** Easy to learn, suitable for system programming, portable.
  • **Disadvantages:** No strict data type checking, debugging can be complex.

Fundamentals of C

  • **Character Set:** The set of alphabets, digits, and symbols used in C.
  • **Comments:** Text in a program that is not compiled or executed, used for information. Single-line comments start with `//`, and multi-line comments are enclosed in `/* ... */`.
  • **C Tokens:** The smallest units in a C program with meaningful representations.
  • **Identifiers and Keywords:** **Identifiers** are names for entities like variables and functions. **Keywords** are reserved words with pre-defined meanings.
  • **Statements:** A set of tokens ending with a semicolon or a curly bracket.
  • **Escape Sequences:** Special character combinations, like `\n` for a new line.
  • **Operators:** Symbols used to perform operations, such as `+`, `-`, `*`, `/`.
  • **Library Functions:** Pre-written functions stored in the C library.

Input/Output Functions

  • `printf()`: A formatted output function.
  • `scanf()`: A function to enter values into a program.

Control Structures in C

These statements change the normal top-to-bottom flow of program execution.

A) Selection/Branching (Conditional)

These statements execute a block of code based on a condition.

if statement

Used for a single condition.


if (condition)
{
    // Block of statements
}

if...else statement

Used for two conditions.


if (condition)
{
    // Statements for true condition
}
else
{
    // Statements for false condition
}

else if statement

Used for multiple conditions.


if (condition 1)
{
    // Statement Block 1
}
else if (condition 2)
{
    // Statement Block 2
}
...
else
{
    // default statement
}

B) Unconditional Statements

These statements break the program flow without a condition.

goto statement

Transfers control to a labeled statement.


label:
// code
goto label;

C) Looping

The process of repeating a block of statements a given number of times or until a condition is met.

for loop

Used when the number of iterations is known.


for (initialization; condition; increment/decrement)
{
    // Statements
}
